Kijelentéslogika (Igazságtáblázatok)

Informatikával kapcsolatos gimnáziumi tanulmányaink kezdeteként Kijelentéslogikával foglalkoztunk. Nulladrendű logika, a Klasszikus Kijelentéslogika témaköre. A logika központi feladata mindig az, hogy meghatározza a helyes következtetés fogalmát. Ezt a klasszikus kijelentéslogikában a szintaxis és a szemantika területén is meg lehet tenni. Viszont a logika feladata az is, hogy ha lehetséges, akkor egy következtetés helyességének eldöntését is leírja. Azaz adjon olyan eljárást/algoritmust, amellyel -- ha lehet -- véges időn belül ki lehessen deríteni, hogy egy következtetés helyes-e vagy sem.

Szintaxis összefoglalása: A ::= p | ¬ A | A ∧ B | A ∨ B | A → B | A ↔ B

Igazságtáblázatok

Az igazságtáblázatokkal megtudhatjuk, hogy ha egy adott logikai kifejezés, egy adott forumulának sorra vesszük minden lehetséges bementét, akkor mik a lehetséges kimenetelek. Egy ilyen logikai táblázatból leolvasható hogy logikai igazsággal állunk-e szemben vagy egy ellentmondással. Hogy helyes-e a következtetés vagy, hogy két kijelentés ekvivalens-e.

Karesz Lore

LogoKaresz - egy Karesz, a Robot oktatóprogram. Ezen típus a Logo típusú feladatokkal készít fel a kásőbbi, bonyolultabb algoritmikus gondolkodást igénylő feladatokra. Először még csak egyszerű ábrákat rajzoltunk mint például egy pókháló, egy-egy sorminta, mint egy lábnyom, de már ezeknek is paraméterezhetőnek kellett mind lennie: állthatónak kellett lennie az ábrák méretének. Ezek után nemsokkal nagyobb fába vágtuk a fejszénk, Mozaikokat kellett rajzolnunk. Eleinte nem volt olyan nehéz. Kellett rajzolni egy cipzárt vagy egy csempét. De lassan jöttek az egy fokkal jobban kinéző mandalák, optikai illúziók, térhatású négyzetes hálók, stb. A Fraktálokba egy kicsit beleizzadtunk, majd jött egy olyan témakör ami még annál is nehezebb volt: rekurzív fák rajzolása.

OOPKaresz

A LogoKaresz témakörrel párhuzamosan vettünk egy másik Karesz, a robot szoftvert is, ami az Objektum Orientált Programozásra(OOP) vezet rá. Ez a program hajaz legjobban az eredeti rendeltetésére a Karesznak. Itt egy négyzethálós pályán kell mozogni Karesszal vagy több Karesszal. Lehetnek azonban akadályok, labirintusok, láva, mozgatható színes kavicsok, melyekre Karesznak vannak különböző érzékelői melyekkel megoldhatja a megadott pályákat, a megadott feladatokat.

Git

A projektjeink, házi feladataink megoldásánál elengedhetetlen volt a Git verziókezelő rendszer használata, mivel házi feladatinkat GitHub-ra feltöltve adhattuk le mentorainknak, tanárainknak. Ezek mellett nekünk is segített a "felhőben" tárolni fájljainkat, hogy hozhassuk-vihessük haza, illetve az iskolába. Visual Stuido 2022 integrációját viszonylag hamar megtanultuk, mivel az volt a legegyszerűbb, majd amikor a Visual Studio Code-ot kezdtük használni Webes projektjeink megoldásához, átvettük a Code integrációját, illetve a GitHub Desktop használatát is, egy bővített Git működés-magyarázat mellett.

Programozási tételek

Ebben a témakörben megtanultuk mik is azok a programozási tételek(mint pl.: megszámlálás, keresés, maximumkeresés, stb.) és 3000+os feladatokkal megtanulhattuk azt, hogy melyek azok a gyakorlati kérdések, amelyekre választ adhatunk ezen tételek alkalmazásával.

Adatszerkezetek

Ebben a témakörben megtanultuk a C#-ban használt leggyakoribb adatszerkezeteinket, azok felhasználási módját. Ezek az adatszerkezetek a következők voltak: szöveg, szám, lista, tömb, szótár, osztály.

HTML, CSS, JS

Ebben a témakörben az interaktív digitális kultúra tankönyv feladataival bevezetést kaptunk a webprogramozás világába. Ezen témakör lezárásáképp készül ez a weboldal, beadandó projekt formában. Számonkérhető tanórai anyagként szerepeli a HTML5 használata, DOM megértése, CSS alapszíntű használata, Google Fonts használata, felhasználói élményt javító JS használata, valamint a szorgalmasabbak számára be lettek mutatva a CSS Framework-ök, leginkább a Tailwind CSS, daisyUI component library alkalmazásával.
